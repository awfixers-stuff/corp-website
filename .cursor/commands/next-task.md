# Next Task

Find `ROADMAP.md` and `TODO.md` (search the entire project root and subdirectories if needed).

1. Read both files completely.
2. Identify the next unfinished item:
   - Prefer items marked as "not started", "todo", unchecked boxes `[ ]`, or the earliest pending entry without a "done"/"completed" marker.
   - If `TODO.md` has a clear prioritized list, use that as the primary source.
   - Cross-reference with `ROADMAP.md` for context or dependencies.
3. Once the next item is identified:
   - Clearly state what the task is.
   - Plan the implementation step-by-step (research codebase first if needed).
   - Execute the work: edit/create files, run tests/build commands, etc.
   - After completion, update `TODO.md` and/or `ROADMAP.md` to mark the item as done (use `[x]` or add a "Completed:" note with date).
4. If no unfinished items remain, report that the roadmap/TODO is complete and suggest reviewing for new items.
5. Always work incrementally and confirm changes before finalizing.

Do not ask for confirmation on every small step — proceed autonomously unless you hit a blocking issue.
